# Modelo de proveedor independiente

Proxi modela a los proveedores como independientes, no como empleados. La estructura del monorepo refleja esa separacion operativa:

- El frontend proveedor contiene la experiencia del proveedor independiente.
- El frontend admin contiene revision, verificacion, moderacion y operacion.
- El backend API centraliza reglas y trazabilidad.

Terminos operativos recomendados:

- proveedor independiente
- verificacion
- oferta
- reserva
- pago protegido
- saldo aprobado
- liquidacion
- reputacion

Esta documentacion es estructural y operativa. No implementa pagos reales, nomina ni cambios fiscales.
